---
title: "JS modules reference"
description: "Complete reference for all JavaScript modules available in Flipper Zero firmware: badusb, gpio, gui, math, notification, serial, storage, and flipper."
---

JS modules extend the built-in functions with access to Flipper Zero hardware and system capabilities. They are written in C/C++ and stored as FAL (Flipper Application File) files on the microSD card.

Load a module with `require()` before calling any of its methods. All loaded modules are automatically unloaded when the script exits.

```javascript
let myModule = require("module_name");
```

<Tip>
  Only load modules you actually use. Each loaded module consumes RAM for the duration of the script.
</Tip>

  <Accordion title="badusb — USB keyboard emulation">
    Enables Flipper Zero to act as a USB HID keyboard, sending key presses and typed strings to a connected computer.

    ```javascript
    let badusb = require("badusb");
    ```

    Call `setup()` before any other method. After setup, check `isConnected()` before sending input.

    **Methods**

    | Method | Description |
    |--------|-------------|
    | `badusb.setup(config?)` | Starts USB HID. Optional config: `vid`, `pid`, `mfrName`, `prodName`, `layoutPath`. |
    | `badusb.isConnected()` | Returns `true` if USB is connected. |
    | `badusb.press(...keys)` | Presses and releases a key or key combo. |
    | `badusb.hold(...keys)` | Holds keys down (up to 5 non-modifier keys simultaneously). |
    | `badusb.release(...keys?)` | Releases held keys. Releases all if called with no arguments. |
    | `badusb.print(str, delay?)` | Types a string. Optional delay (ms) between key presses. |
    | `badusb.println(str, delay?)` | Same as `print()`, followed by pressing ENTER. |
    | `badusb.altPrint(str, delay?)` | Types using Alt+Numpad method. Windows only. |
    | `badusb.altPrintln(str, delay?)` | Same as `altPrint()`, followed by pressing ENTER. |
    | `badusb.quit()` | Releases USB HID, allowing interchange with other USB modes. |

    **Key names**

    Modifier keys: `CTRL`, `SHIFT`, `ALT`, `GUI`

    Special keys: `DOWN`, `LEFT`, `RIGHT`, `UP`, `ENTER`, `DELETE`, `BACKSPACE`, `END`, `HOME`, `ESC`, `INSERT`, `PAGEUP`, `PAGEDOWN`, `CAPSLOCK`, `NUMLOCK`, `SCROLLLOCK`, `PRINTSCREEN`, `PAUSE`, `SPACE`, `TAB`, `MENU`, `F1`–`F24`, `NUM0`–`NUM9`

    You can also pass a numeric HID key code directly.

    **Examples**

    ```javascript
    let badusb = require("badusb");

    // Start with default USB parameters
    badusb.setup();

    // Start with custom VID:PID and device name
    badusb.setup({ vid: 0xAAAA, pid: 0xBBBB, mfrName: "Flipper Devices", prodName: "Flipper Zero" });

    if (badusb.isConnected()) {
        badusb.println("Hello, world!"); // Type text and press Enter
        badusb.press("CTRL", "a");       // Select all
        badusb.press("CTRL", "SHIFT", "ESC"); // CTRL+SHIFT+ESC combo
        badusb.press(0x47);              // Scroll Lock (HID code 0x47)
        badusb.hold("CTRL", "v");        // Hold CTRL+V
        badusb.release();                // Release all held keys
    }
    ```
  </Accordion>

  <Accordion title="gpio — GPIO pin control">
    Controls the GPIO pins on Flipper Zero's expansion connector. Supports digital input/output, analog input, interrupts, and PWM.

    This module depends on `event_loop`, which must be imported first.

    ```javascript
    let eventLoop = require("event_loop");
    let gpio = require("gpio");
    ```

    **Methods**

    | Method | Description |
    |--------|-------------|
    | `gpio.get(pin)` | Returns a `Pin` object for the given pin identifier (e.g. `"pc3"`, `"pa6"`, `7`). |

    **Pin object methods**

    | Method | Description |
    |--------|-------------|
    | `pin.init(mode)` | Configures the pin. See mode options below. |
    | `pin.write(value)` | Writes a boolean logic level to an output pin. |
    | `pin.read()` | Reads a boolean logic level from a digital input pin. |
    | `pin.readAnalog()` | Reads an analog voltage in millivolts from an analog input pin. |
    | `pin.interrupt()` | Attaches an interrupt; returns an event loop `Contract`. |
    | `pin.isPwmSupported()` | Returns `true` if this pin supports PWM. |
    | `pin.pwmWrite(freq, duty)` | Starts PWM with the given frequency (Hz) and duty cycle (%). |
    | `pin.isPwmRunning()` | Returns `true` if PWM is currently running on this pin. |
    | `pin.pwmStop()` | Stops PWM on this pin. |

    **Mode configuration**

    ```javascript
    // Output: push-pull
    pin.init({ direction: "out", outMode: "push_pull" });

    // Output: open-drain
    pin.init({ direction: "out", outMode: "open_drain" });

    // Input: digital
    pin.init({ direction: "in", inMode: "plain_digital" });

    // Input: analog
    pin.init({ direction: "in", inMode: "analog" });

    // Input: interrupt on rising edge
    pin.init({ direction: "in", inMode: "interrupt", edge: "rising" });
    // edge options: "rising", "falling", "both"
    // pull options: "up", "down" (optional)
    ```

    **Example**

    ```javascript
    let eventLoop = require("event_loop");
    let gpio = require("gpio");

    let led = gpio.get("pc3");
    led.init({ direction: "out", outMode: "push_pull" });

    led.write(true);
    delay(1000);
    led.write(false);
    delay(1000);
    ```

  <Accordion title="gui — graphical user interface">
    Provides access to Flipper Zero's display and input system through a ViewDispatcher. GUI requires `event_loop` and must be imported after it.

    ```javascript
    let eventLoop = require("event_loop");
    let gui = require("gui");
    ```

    The `gui` module itself manages the `viewDispatcher`, which switches between views and handles input events. Individual views are loaded as submodules (see below).

    **ViewDispatcher API**

    | Method / Property | Description |
    |-------------------|-------------|
    | `gui.viewDispatcher.switchTo(view)` | Switches to the given `View`. |
    | `gui.viewDispatcher.sendTo(direction)` | Sends the viewport to `"front"` or `"back"`. |
    | `gui.viewDispatcher.sendCustom(event)` | Fires a custom numeric event. |
    | `gui.viewDispatcher.custom` | Event source `Contract` for custom events. |
    | `gui.viewDispatcher.navigation` | Event source `Contract` fired when the back key is pressed. |
    | `gui.viewDispatcher.currentView` | The `View` currently being shown. |

    **ViewFactory methods** (returned by each view submodule)

    | Method | Description |
    |--------|-------------|
    | `ViewFactory.make()` | Creates a `View` instance with default properties. |
    | `ViewFactory.makeWith(props, children?)` | Creates a `View` and sets initial properties. |

    **View methods**

    | Method | Description |
    |--------|-------------|
    | `view.set(property, value)` | Sets a view property by name. |
    | `view.addChild(child)` | Adds a child element to the view. |
    | `view.resetChildren()` | Removes all children from the view. |
    | `view.setChildren(children)` | Replaces all children with a new array. |

    **Multi-view example**

    ```javascript
    let eventLoop = require("event_loop");
    let gui = require("gui");
    let loadingView = require("gui/loading");
    let submenuView = require("gui/submenu");
    let emptyView = require("gui/empty_screen");

    let views = {
        loading: loadingView.make(),
        empty: emptyView.make(),
        demos: submenuView.makeWith({
            items: ["Hourglass screen", "Empty screen", "Exit app"],
        }),
    };

    eventLoop.subscribe(views.demos.chosen, function (_sub, index, gui, eventLoop, views) {
        if (index === 0) {
            gui.viewDispatcher.switchTo(views.loading);
        } else if (index === 1) {
            gui.viewDispatcher.switchTo(views.empty);
        } else if (index === 2) {
            eventLoop.stop();
        }
    }, gui, eventLoop, views);

    eventLoop.subscribe(gui.viewDispatcher.navigation, function (_sub, _, gui, views) {
        gui.viewDispatcher.switchTo(views.demos);
    }, gui, views);

    gui.viewDispatcher.switchTo(views.demos);
    eventLoop.run();
    ```

    ### gui/dialog

    Displays a dialog with a header, body text, and up to three labeled buttons.

    ```javascript
    let dialogView = require("gui/dialog");
    ```

    **Props**

    | Prop | Type | Description |
    |------|------|-------------|
    | `header` | `string` | Bold text at the top of the screen. |
    | `text` | `string` | Text in the middle of the screen. |
    | `left` | `string` | Label for the left button. Hidden if unset. |
    | `center` | `string` | Label for the center button. Hidden if unset. |
    | `right` | `string` | Label for the right button. Hidden if unset. |

    **Events**

    | Event | Type | Description |
    |-------|------|-------------|
    | `input` | `string` | Fires on button press. Value is `"left"`, `"center"`, or `"right"`. |

    ```javascript
    let dialog = dialogView.makeWith({
        header: "Confirm",
        text: "Continue?",
        left: "No",
        right: "Yes",
    });

    eventLoop.subscribe(dialog.input, function (_sub, button, eventLoop) {
        print("Pressed:", button);
        eventLoop.stop();
    }, eventLoop);

    gui.viewDispatcher.switchTo(dialog);
    eventLoop.run();
    ```

    ### gui/file_picker

    Prompts the user to select a file from the filesystem. Unlike other GUI views, this is a function call, not a view.

    ```javascript
    let filePicker = require("gui/file_picker");
    ```

    **`filePicker.pickFile(basePath, extension)`**

    | Parameter | Description |
    |-----------|-------------|
    | `basePath` | The directory to open. |
    | `extension` | File extension filter, e.g. `".sub"`, `".iso|.img"`, or `"*"` for all files. |

    Returns a `string` path to the selected file, or `undefined` if the user cancelled.

    ```javascript
    let filePicker = require("gui/file_picker");

    let path = filePicker.pickFile("/ext", ".sub");
    if (path !== undefined) {
        print("Selected:", path);
    } else {
        print("Cancelled");
    }
    ```

    ### gui/widget

    Displays a combination of custom graphical elements on one screen.

    ```javascript
    let widgetView = require("gui/widget");
    ```

    This view has no props. Instead, it takes children — elements defined as objects with an `element` type field.

    **Element types**

    | Element | Key properties | Description |
    |---------|---------------|-------------|
    | `string` | `x`, `y`, `align`, `font`, `text` | Single-line text. |
    | `string_multiline` | `x`, `y`, `align`, `font`, `text` | Multi-line text. |
    | `text_box` | `x`, `y`, `w`, `h`, `align`, `text`, `stripToDots` | Scrollable text box area. |
    | `text_scroll` | `x`, `y`, `w`, `h`, `text` | Vertically scrollable text. |
    | `button` | `text`, `button` | Button at the bottom (`"left"`, `"center"`, or `"right"`). |
    | `icon` | `x`, `y`, `iconData` | Displays an icon (see `gui/icon`). |
    | `rect` | `x`, `y`, `w`, `h`, `radius`, `fill` | Rectangle, optionally rounded and filled. |
    | `circle` | `x`, `y`, `radius`, `fill` | Circle, optionally filled. |
    | `line` | `x1`, `y1`, `x2`, `y2` | Line between two points. |

    **`button` event**

    | Event | Type | Description |
    |-------|------|-------------|
    | `button` | `ButtonEvent` | Fires when a button is pressed. Fields: `key` (`"left"`, `"center"`, `"right"`) and `type` (`"press"`, `"release"`, `"short"`, `"long"`, `"repeat"`). |

    ```javascript
    let widget = widgetView.makeWith({}, [
        { element: "string", x: 64, y: 10, align: "cm", font: "primary", text: "Hello!" },
        { element: "button", button: "right", text: "Next" },
        { element: "button", button: "left", text: "Back" },
        { element: "rect", x: 10, y: 20, w: 108, h: 30, radius: 3, fill: false },
    ]);

    eventLoop.subscribe(widget.button, function (_sub, event, eventLoop) {
        print("Button:", event.key, "Type:", event.type);
        if (event.type === "short") eventLoop.stop();
    }, eventLoop);
    ```

  <Accordion title="serial — UART serial communication">
    Transmits and receives data over Flipper Zero's UART interfaces (`usart` or `lpuart`).

    ```javascript
    let serial = require("serial");
    ```

    Call `setup()` before any other method. Calling `setup()` automatically disables the Expansion module service to prevent interference.

    **Methods**

    | Method | Description |
    |--------|-------------|
    | `serial.setup(port, baud, framing?)` | Configures the serial port. `port`: `"usart"` or `"lpuart"`. Optional `framing` object: `{ dataBits, parity, stopBits }`. |
    | `serial.write(...data)` | Writes data. Accepts strings, numbers (single byte), arrays of numbers, `ArrayBuffer`, or `DataView`. |
    | `serial.read(n, timeout?)` | Reads exactly `n` bytes. Returns a string or `undefined` on timeout. |
    | `serial.readln(timeout?)` | Reads until a line break. Returns a string or `undefined` on timeout. |
    | `serial.readAny(timeout?)` | Reads any available data. Returns a string or `undefined` on timeout. |
    | `serial.readBytes(n, timeout?)` | Reads `n` bytes. Returns an `ArrayBuffer` or `undefined` on timeout. |
    | `serial.expect(patterns, timeout?)` | Waits for a string or byte pattern. Returns the index of the matched pattern or `undefined`. |
    | `serial.end()` | Deinitializes the serial port (allows re-initialization within the same script). |

    **Framing configuration**

    | Option | Values |
    |--------|--------|
    | `dataBits` | `"6"`, `"7"`, `"8"`, `"9"` |
    | `parity` | `"none"`, `"even"`, `"odd"` |
    | `stopBits` | `"0.5"`, `"1"`, `"1.5"`, `"2"` (LPUART only supports `"1"` and `"2"`) |

    ```javascript
    let serial = require("serial");

    serial.setup("lpuart", 115200);

    serial.write("AT\r\n");
    let response = serial.readln(1000);
    if (response !== undefined) {
        print("Response:", response);
    }

    // Wait for one of two patterns
    let idx = serial.expect([": not found", "Usage: "]);
    print("Matched pattern index:", idx);

    serial.end();
    serial.setup("usart", 9600);
    serial.write(0x0a); // Write a single byte
    serial.write("Hello, world!", [0x0d, 0x0a]); // String + CRLF
    ```
  </Accordion>

  <Accordion title="storage — file and directory access">
    Reads and writes files and directories on the Flipper Zero filesystems: `/int/` (internal flash) and `/ext/` (microSD card).

    ```javascript
    let storage = require("storage");
    ```

    **File paths**

    Paths use `/int/` for internal flash or `/ext/` for the microSD card:

    ```
    /ext/example_dir/example_file.txt
    ```

    **File class** (returned by `openFile()`)

    | Method | Description |
    |--------|-------------|
    | `file.isOpen()` | Returns `true` if the file is open. |
    | `file.read(mode, bytes)` | Reads bytes. `mode`: `"ascii"` returns a string; `"binary"` returns `ArrayBuffer`. |
    | `file.write(data)` | Writes a string or `ArrayBuffer`. Returns bytes written. |
    | `file.seekRelative(bytes)` | Moves R/W pointer forward by `bytes`. |
    | `file.seekAbsolute(bytes)` | Moves R/W pointer to an absolute position. |
    | `file.tell()` | Returns current R/W pointer position. |
    | `file.truncate()` | Discards data after the current pointer position. |
    | `file.size()` | Returns the file size in bytes. |
    | `file.eof()` | Returns `true` if the pointer is at end of file. |
    | `file.copyTo(dest, bytes)` | Copies bytes from this file into another open file. |
    | `file.close()` | Closes the file. |

    **Module methods**

    | Method | Description |
    |--------|-------------|
    | `storage.openFile(path, access, openMode)` | Opens a file. `access`: `"r"`, `"w"`, `"rw"`. `openMode`: `"open_existing"`, `"open_always"`, `"open_append"`, `"create_new"`, `"create_always"`. Returns a `File` or `undefined`. |
    | `storage.fileExists(path)` | Returns `true` if a file exists at `path`. |
    | `storage.fileOrDirExists(path)` | Returns `true` if a file or directory exists. |
    | `storage.directoryExists(path)` | Returns `true` if a directory exists. |
    | `storage.stat(path)` | Returns a `FileInfo` struct (`path`, `isDirectory`, `size`, `accessTime`) or `undefined`. |
    | `storage.fsInfo(filesystem)` | Returns a `FsInfo` struct (`totalSpace`, `freeSpace`) or `undefined`. |
    | `storage.makeDirectory(path)` | Creates an empty directory. |
    | `storage.readDirectory(path)` | Returns an array of `FileInfo` entries or `undefined`. |
    | `storage.copy(src, dst)` | Copies a file or directory recursively. |
    | `storage.rename(src, dst)` | Renames or moves a file or directory. |
    | `storage.remove(path)` | Removes a file or empty directory. |
    | `storage.rmrf(path)` | Recursively removes a file or non-empty directory. |
    | `storage.arePathsEqual(p1, p2)` | Returns `true` if both paths refer to the same location. |
    | `storage.isSubpathOf(parent, child)` | Returns `true` if `child` is a subpath of `parent`. |
    | `storage.nextAvailableFilename(dir, name, ext, maxLen)` | Returns the next available filename with a numeric suffix. |

    ```javascript
    let storage = require("storage");

    // Write a file
    let file = storage.openFile("/ext/hello.txt", "w", "create_always");
    file.write("Hello, Flipper!");
    file.close();

    // Read the file back
    let f = storage.openFile("/ext/hello.txt", "r", "open_existing");
    let content = f.read("ascii", 100);
    print(content); // "Hello, Flipper!"
    f.close();

    // Check free space on the SD card
    let info = storage.fsInfo("/ext");
    print("Free space:", info.freeSpace, "bytes");

    // List directory contents
    let entries = storage.readDirectory("/ext/apps");
    if (entries !== undefined) {
        for (let i = 0; i < entries.length; i++) {
            print(entries[i].path, entries[i].isDirectory ? "(dir)" : "");
        }
    }

    // Clean up
    storage.remove("/ext/hello.txt");
    ```
